    smart point:
    auto_pointer

    c++11以前的auto pointer (现已弃用)，实现了自动释放动态内存

    1.存在复制或者赋值会改变所有权

    p1会先析构原先管理的内存，然后管理p2所指的内存，p2则为NULL,失去所有权

    2.不支持对象数组的内存管理

    unique_pointer

    c++11后取代auto_pointer

    特性：1.两指针不能指向同一内存。 2.无法进行左值的赋值操作，要使用move转化为右值进行操作，使程序员知道后果

    shared_pointer
    weak_pointer

    辅助share_pointer 使用，引用不会增加use.count()，不支持->,和*

    string_view

    一个字符串视图，不拥有字符数据，引用string等类型的数据，有利于高效传递字符串数据

    某种程度上可以理解为一个封装指针，但具有更多功能以及安全性

    cmu15445:

    project 0 c++ primer:

    **explicit:**用于单参数构造函数，防止隐式类型转化引入bug,更加安全的使用类

    隐式类型转换：Implicit conversions are performed whenever an expression of some type T1 is used in context that does not accept that type, but accepts some other type T2,即就是那些编译器自己转换类型，显式就是coder自己在代码中实现的。

    future:std::The class template std::future provides a mechanism to access the result of asynchronous operations:中用于获取异步操作结果的模板类，与多线程相关.

    右值引用：

    右值和左值：有地址的变量就是左值，没有地址的字面值、临时值就是右值。

    右值引用：引用右值，用&&，可以引用左值通过std::move()(实际上可以看成是一种类型转换)

    左值引用：引用左值，用&(常见的，e.g 深拷贝)，也可以引用右值通过const 左值引用,(const int &ref_a = 5;)

    应用：实现语义移动（将对象的资源所有权进行转让）当一个临时对象（右值）将要销毁时，它的资源可以被移动到另一个对象，提高性能。移动构造函数和移动赋值运算符

    delete:

    禁止拷贝构造函数使用,用于一般函数删除某些函数的重载

    **noexcept:**用于声明这个函数不会抛出任何错误，提高性能

    default:标记默认构造函数（C++11)

    shared_ptr:

    属于四大smart pointer ,本质上是一个类，和原始指针（一种基本数据类型）不同

    可以用多个指针管理同一块内存，类内部维护一个计数，表示引用个数，当为0时被析构（防止内存泄漏）
